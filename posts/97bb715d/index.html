前言

今天我們要開始寫我們的第一個 Activity。我們要來認識一下什麼是 Activity。

  • 什麼是 Activity
  • Activity 的架構
  • 綁定設計

什麼是 Activity

Activity 就是我們每一個 app 的頁面，每個頁面都會自己Activity，我們可以透過撰寫不同 Activity 來展現不同的功能，同時每個 Activity 之間也能透過 Intent 相互溝通。

我們會利用 java 或是 kotlin 來撰寫我們的 Activity。我們的 Activity 會放在 java 資料夾下(kotlin 也是)。


Activity 的架構

我們在一開始創立 project 的時候會出現下面這些，其實我們可以觀察到，我們的 MainActivity 是繼承自 AppCompatActivity。

class MainActivity : AppCompatActivity() {
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)
    }
}

如果我們到 Android documentation 裡面，我們可以看到 Activity 的階層關係:



onCreate

onCreate 是在我們 Activity 剛創建時要做的事情。我們需要去覆寫 onCreate。例如我們需要將變數跟我們的外觀設計綁定或是我們有一些屬性就需要在最一開始處理，例如我們看到：

setContentView(R.layout.activity_main)

這行的用意就在將我們的 Activity 與某個 layout 綁定。這邊我們是綁定 activity_main.xml，假如我們今天有另一個 layout 叫做 hello_main.xml，則我們也可以改成：

setContentView(R.layout.hello_main)

事實上，我們 onCreate 跟我們 Activity 的生命週期有關，但是我們這邊就先不說明，我們留待之後做介紹，這邊我先把關係圖放出來讓大家有個印象。


綁定設計

我們剛剛在講 onCreate 時，提到過我們需要綁定物件。綁定物件的用意式在之後能讓程式知道我們在使用哪個物件。因此我們需要在 layout 設定 ID，這個 ID 需要是唯一（對同一個 layout file）。 這邊我們創立一個 EditText，xml 程式為：

<?xml version="1.0" encoding="utf-8"?>
<androidx.constraintlayout.widget.ConstraintLayout
    xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:app="http://schemas.android.com/apk/res-auto"
    xmlns:tools="http://schemas.android.com/tools"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    tools:context=".MainActivity">

    <EditText
        android:id="@+id/edt_name"
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:ems="10"
        android:inputType="textPersonName"
        android:text="Name"
        app:layout_constraintBottom_toBottomOf="parent"
        app:layout_constraintEnd_toEndOf="parent"
        app:layout_constraintStart_toStartOf="parent"
        app:layout_constraintTop_toTopOf="parent" />
        
</androidx.constraintlayout.widget.ConstraintLayout>

注意我們在 EditText 裡加入一行

android:id="@+id/edt_name"

這行就是在為我們的 EditText 加入 ID，前面的 @+id 是固定的，後面的才是我們真正的名字。一般我們命名的規則就是 <views_name>_<usage>，例如我們是 EditText 並且用來輸入名字，所以就叫做 edt_name。

在我們 onCreate 時我們想要綁定這個物件時我們需要輸入：

val edt_name = findViewById<EditText>(R.id.edt_name)

R 就是在編譯應用程式後所產生的 R 類別。此類別內含 res/ 目錄中所有資源的 ID。因此我們可以利用 R.id 來存取我們的 ID。

此時我們直接讓 edt_name 的文字顯示成 Hello World，我們可以這樣做：

edt_name.setText("Hello Wrold")

完整的程式碼會是：

package com.example.myapplication

import androidx.appcompat.app.AppCompatActivity
import android.os.Bundle
import android.widget.EditText

class MainActivity : AppCompatActivity() {
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)

        val edtName = findViewById<EditText>(R.id.edt_name)
        edtName.setText("Hello World")
    }
}

當我們一打開我們的手機他就會顯示 Hello World !

其實我們在 Kotlin 不需要去綁定我們的物件就可以使用了！也就是我們可以單純用物件的 ID 來執行

edt_name.setText("Hello World")

至於為什麼可以這樣做是因為在我們的 build.gradle 裡有三行程式碼：

apply plugin: 'com.android.application'
apply plugin: 'kotlin-android'
apply plugin: 'kotlin-android-extensions'

這三行的意思是用來加入 plugin ，來讓我們寫程式的時候能更方便。 可以參考這篇。
