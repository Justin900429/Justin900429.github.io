前言

什麼是 Android?

Android 是基於 Linux kernel 開發的手機平板作業系統(Operating system)。Android 最初是由 Anroid Inc. 開發，最後由 Google 於 2005 收購。現在已經成為開放原始碼，加快 Android 的普及。下圖是 Android 的架構，最上層為使用者所使用的應用程式。

Android architecture

我們來認識一下 Android 的架構，更理解他是怎麼運作的。下面是更清楚的表：


從上表知道我們總共有五個層，六個區塊：

  1. Linux Kernel
  2. Hardware abstraction layer
  3. Libraries(Native C/C++), Android runtime
  4. Application framework (API - Java or Kotlin)
  5. System applications

Linux Kernel

Android 利用 Linux kernel 來實現許多 Linux 的函式，例如 Thread, Memory management 等等，同時也可以使 Linux kernel 的工程師更好的開發 Android。

Hardware abstraction layer (HAL)

這層利用許多的函式庫以及介面來連接硬體與 Application framework。對於每一個硬體組成都會用各自的介面來連接。

Libraries(Native C/C++)

這裡收錄了很多必要的底層函數供 Application framework 或是 Android runtime 使用

Android runtime(ART)

ART 會為每一個執行的 app 建立一個 process，來管理這個 app 的生命週期。

Application framework (API - Java or Kotlin)

這層是用來開發手機 app 的地方，開發者不須了解如何背後如何實現這些功能，相反，使用者只需要用 API 來做到功能的實現。

System applications

使用者所使用的是程式。
