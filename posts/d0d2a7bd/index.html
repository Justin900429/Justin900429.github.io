前言

上篇我們說明 Android 系統，這篇我們要來講解 Android 的開發環境。

  • Android Studio
  • 開發流程
  • Android 專案架構

Android Studio

我們一般開發都是利用 Android studio 來建立我們的應用程式。 Android studio 整合了以下的工具（下載連結)：

這些工具在下載 Android studio 時一般都會配備完成，但是 Java SE 可能要另外安裝。

1. Android SDK

SDK 代表 Software Development Kit，是在軟體開發很好用的工具。裡頭包含 Android simulator、Debuging、Monitoring 等等。

2. Java JDK

撰寫 Java 所需要的環境，也就是 Java development kit。

3. Plugins

一般在使用外部工具可安裝的軟件，例如 Firebase，Flutter 等等。

開發流程

一般來說我們會有一個開發流程供參考，這樣我們就可以知道我們要怎麼做：



每個團隊的開發流程可能會有一些不一樣的地方，但是整體來說都是一樣的。最複雜的地方莫過於 debugging。這步可能會花很多的時間做，因此平時寫程式就要養成良好的習慣，才不會花費過多力氣在 debugging。

創建 project

當我們打開 Android studio 時，會出現：

我們點選 Start a new Android Studio project 之後，會出現選擇模板的頁面，這邊我們只是要介紹環境，因此我們選擇 Empty Activity。

接著我們可以輸入我們的專案名，這邊我就不改動。Package name 有點類似我們專案的 id，之後上架也是看 package name。位置就看讀者需要放置，語言設置為 Kotlin，Minimum SDK 指的就是這 app 最低支援到第幾版的 Android。

建置完成後，系統會幫我們創立兩個檔案，分別是 <your_application_name>.kt 跟 activity_main.xml。 xml 檔案負責呈現排版，而 kt 則是用作創立函式，設計監聽，轉換頁面等等功能。

Android 專案架構

左邊的 Resource manager 會出現下表的檔案以及資料：

Gradle Scripts

這是用來記錄這個專案全部所需的環境，以及外部的資料以及函式庫。每當有改變到 gradle 的設定時，需重新建置環境。 另外 gradle 的語法是 Maven。

Manifests

這個資料夾裡面有一個 manifest.xml，這個檔案記錄了這個專題的重要資訊，例如專案名稱、Package name、權限、icon、Activity 等等。

例如下面就記錄我的 Activity 的資訊。

<activity android:name=".MainActivity">
    <intent-filter>
        <action android:name="android.intent.action.MAIN" />

        <category android:name="android.intent.category.LAUNCHER" />
    </intent-filter>
</activity>

java

這邊是管理 code 的部分，可以包含 Class、Activity 等等。而他下面的命名方式會跟我們的 package name 有關，例如我剛剛取 com.example.myapplication，則我的 Activity 會產生在名為 com.example.myapplication 的資料夾下。



res

這是用來管理各種不同的資源的資料夾



  • drawable
    用來存放照片的地方。
  • layout
    存放設定外觀的 xml
  • mipmap
    存放 app icon 的地方，裡頭不同資料夾代表不同的解析度。
  • values
    用來設定外觀的樣式，可以被多次重複利用。例如同樣的檔案可以設定按鈕顏色的漸層或背景的漸層等等，
